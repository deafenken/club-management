## 二、5分钟答辩讲解稿

### 开场（成员A，1分钟）
"各位老师好，我们小组做的是面向高校社团的活动组织与资源管理系统。

高校社团管理长期面临三个痛点：第一，信息分散——通知靠微信群、审批靠纸质单、统计靠Excel；第二，审批流程冗长——一个活动要跑三个部门；第三，活动策划效率低——社长写方案靠经验、没有模板。

我们的系统解决这些问题的同时，还做了一项差异化创新——集成了AI大模型，可以智能生成完整的活动策划方案。

接下来我们分模块介绍。"

### 模块1：社团与活动管理（成员B，1分钟）
"我负责社团基础管理和活动全流程。系统有4类角色：管理员、指导老师、社长和学生，权限通过RBAC模型隔离。

活动的生命周期用状态机控制：草稿→提交审批→管理员审批通过→学生报名→活动当天签到→活动结束。报名有人数上限控制，签到用6位随机码防伪。这个流程覆盖了社团活动从策划到执行再到归档的完整生命周期。"

### 模块2：资源管理（成员C，1分钟）
"我负责场地、物资、经费三个资源模块。场地预约的核心难点是冲突检测——同一场地同一时段不能同时分配给两个活动，我的实现是在数据库层面联合查询，发现冲突会拒绝申请并提示已被占用。

物资管理有库存概念，借用扣库存、归还恢复。经费管理覆盖预算申请到报销跟踪的完整链路。三个模块打通了社团资源申请的全流程。"

### 模块3：AI智能策划—创新亮点（成员D，1.5分钟）⭐
"我负责系统的创新模块——AI智能活动策划。

用户输入活动需求后，系统通过精心设计的System Prompt调用DeepSeek大模型，自动生成包含活动概述、流程安排、资源需求、预算明细、宣传方案、应急预案的完整策划方案。整个方案以结构化的Markdown格式输出，直接可以在前端渲染展示。

## 三、高频答辩问题+标准答案

### Q1：你们和直接调ChatGPT有什么区别？
**答（成员D）**：直接调ChatGPT的问题是输出格式不受控，经常偏离主题。我们的方案有三层优化：
1. 结构化Prompt——精心设计的System Prompt强制AI按固定章节（活动概述、流程、资源、预算、宣传、应急方案）输出完整策划方案
2. 双Provider架构——同时支持DeepSeek和Anthropic Claude，通过配置即可切换，不绑定单一厂商
3. 全链路追踪——每次调用记录到llm_call_log表，包含耗时、状态、错误信息，可事后审计和优化

### Q2：Mermaid图表工具类是怎么设计的？
**答（成员D）**：虽然是目前AI模块主要生成Markdown策划方案，但我们预留了MermaidUtil工具类用于未来扩展图表生成。它做了4步清洗：
1. 正则提取```mermaid代码块
2. 12组中文标点→英文标点批量替换
3. 非法Unicode字符过滤
4. 自动计数并补全缺失的闭合花括号
预校验逻辑：gantt必须有title和section，erDiagram必须有实体定义。这个工具类是通用的，后续任何需要从LLM输出中提取Mermaid代码的场景都能复用。

### Q3：场地冲突检测怎么实现的？
**答（成员C）**：SQL联合查询。条件：同一场地ID、同一日期、申请的开始时间早于已有预约的结束时间 且 申请结束时间晚于已有预约的开始时间、已有预约状态为APPROVED。如果有结果就是冲突。这是标准的区间重叠检测算法。

### Q4：如果AI API挂了怎么办？
**答（成员D）**：三层保障。第一层，OkHttp客户端设置了30秒连接超时和120秒读取超时。第二层，最多3次重试，间隔递增（2s→4s→8s），指数退避。第三层，所有调用记录写入llm_call_log表，可以事后追溯和补调。另外，系统其他所有功能（社团管理、活动管理、场地预约、物资借用）完全不依赖AI模块，AI不可用时不影响核心业务。

### Q5：四个人怎么分工的？工作量均衡吗？
**答（成员A）**：非常均衡。A做权限体系+社团管理，B做活动全流程+数据大屏，C做资源管理三模块，D做AI策划+LLM全链路。每个人的代码量在600-800行Java/Vue，加上Prompt设计和配置文件。我们用Git分支管理，API文档先行，每人独立开发自己的模块。

### Q6：安全性怎么保证的？
**答（成员A）**：密码BCrypt加密存储、JWT无状态认证2小时过期、Spring Security接口级RBAC权限拦截、MyBatis参数绑定防SQL注入。前端路由守卫+后端接口拦截双重校验。

### Q7：和市面上的社团管理工具有什么不同？
**答（成员D）**：市面上的工具基本都是纯信息管理——CRUD社团、活动、成员。我们的差异化是AI智能策划——输入一段活动需求描述，系统自动生成包含活动概述、流程安排、资源需求、预算明细、宣传方案、应急预案的完整策划方案文档。这个AI辅助策划能力在同类系统中较为少见。

## 五、项目部署说明

### 环境要求
- JDK 8+、Maven 3.6+、Node.js 18+、MySQL 8.0
- DeepSeek API Key（从 https://platform.deepseek.com 获取）

### 启动步骤

**1. 导入数据库**
```bash
mysql -u root -p < D:\club-management\sql\init.sql
```

**2. 配置LLM API Key**
编辑 `backend/src/main/resources/application.yml`：
```yaml
llm:
  api:
    key: sk-your-deepseek-api-key-here
```

**3. 启动后端**
```bash
cd D:\club-management\backend
mvn spring-boot:run
# 访问 http://localhost:8080
```

**4. 启动前端**
```bash
cd D:\club-management\frontend
npm install
npm run dev
# 访问 http://localhost:5173
```

**5. 登录**
| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | admin | 123456 |
| 社长 | president01 | 123456 |
| 学生 | student01 | 123456 |

### 如果没有 DeepSeek API Key（答辩备用方案）
可以将 `application.yml` 中 `llm.api.key` 设为任意值。系统会在调用AI时返回错误提示，但其他所有功能（社团管理、活动管理、场地预约、物资借用、数据大屏）完全正常运行。演示时可以展示AI智能策划页面的模板和交互流程。
