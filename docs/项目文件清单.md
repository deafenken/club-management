# 高校社团管理系统 — 项目文件清单

> 三层架构：**前端** (Vue3 + Vite) → **后端** (SpringBoot + MyBatis-Plus) → **数据库** (MySQL)

---

## 🔵 一、前端（Frontend）— `frontend/`

**技术栈**：Vue 3.4 + Vite 5.4 + Element Plus 2.7 + ECharts 5.5 + Pinia 2.1 + Vue Router 4.3 + Axios 1.7
**启动方式**：`cd frontend && npx vite --host` → 端口 **5173**

### 1.1 配置文件

| 文件 | 作用 |
|------|------|
| `frontend/package.json` | 项目依赖声明 (18个依赖 + 2个开发依赖) |
| `frontend/package-lock.json` | 依赖版本锁定文件 |
| `frontend/vite.config.js` | Vite 构建配置：端口5173 + `/api` 代理到 `localhost:8080` |
| `frontend/index.html` | HTML 入口：`<div id="app">` + 暗黑模式防闪烁脚本 |

### 1.2 框架核心

| 文件 | 作用 |
|------|------|
| `frontend/src/main.js` | 应用入口：创建 Vue 实例、注册 Pinia/Router/ElementPlus/图标 |
| `frontend/src/App.vue` | 根组件：`<router-view>` 路由出口 + 页面过渡动画 + NProgress 进度条 |

### 1.3 路由

| 文件 | 作用 |
|------|------|
| `frontend/src/router/index.js` | 路由表（10条路由）+ 导航守卫（未登录→跳转登录页） |

### 1.4 API 层

| 文件 | 作用 |
|------|------|
| `frontend/src/api/request.js` | Axios 实例：baseURL=`/api`、JWT 自动携带、401 自动跳转登录、统一错误提示 |

### 1.5 全局样式

| 文件 | 作用 |
|------|------|
| `frontend/src/styles/global.scss` | 全局样式：CSS 变量（主题色#1890ff）、暗黑模式变量、滚动条美化、Element Plus 组件微调 |

### 1.6 页面组件（10个）

| 文件 | 对应路由 | 功能说明 |
|------|----------|----------|
| `frontend/src/views/Login.vue` | `/login` | 登录/注册页：JWT 认证、表单验证、渐变背景+浮动装饰 |
| `frontend/src/views/Layout.vue` | `/` (父路由) | 主布局：白色侧边栏+蓝白顶栏+折叠/主题切换/用户信息/退出 |
| `frontend/src/views/Dashboard.vue` | `/` (首页) | 数据看板：4个统计卡片 + ECharts 饼图(活动状态) + 柱状图(社团排名) + 公告列表 |
| `frontend/src/views/ClubList.vue` | `/clubs` | 社团管理：社团卡片列表+搜索+创建社团+社长审批加入申请 |
| `frontend/src/views/ActivityList.vue` | `/activities` | 活动管理：活动表格+发布活动+社团选择+报名+签到码输入 |
| `frontend/src/views/VenueList.vue` | `/venues` | 场地预约：场地卡片+时间选择+冲突检测+预约提交 |
| `frontend/src/views/ResourceList.vue` | `/resources` | 物资管理：物资表格+库存显示+借用申请+数量/归还日期 |
| `frontend/src/views/FundList.vue` | `/funds` | 经费管理：经费表格+状态筛选+收入/支出申请+社团选择 |
| `frontend/src/views/AiPlan.vue` | `/ai-plan` | AI 智能策划：输入活动需求 → DeepSeek 生成策划方案 (Markdown渲染) |
| `frontend/src/views/AdminPage.vue` | `/admin` | 系统管理（管理员）：5个Tab — 社团审批+活动审批+场地预约审批+物资借用审批+经费审批+用户管理 |

## 🟢 二、后端（Backend）— `backend/`

**技术栈**：SpringBoot 2.7 + MyBatis-Plus 3.5 + SpringSecurity + JWT(0.9.1) + Hutool 5.8
**启动方式**：`cd backend && mvn spring-boot:run` → 端口 **8080**
**要求**：Java 17, Maven 3.9, 环境变量 `DEEPSEEK_API_KEY`

### 2.1 构建配置

| 文件 | 作用 |
|------|------|
| `backend/pom.xml` | Maven 项目定义：SpringBoot 2.7.18、MyBatis-Plus、MySQL驱动、JWT、Hutool、Lombok |

### 2.2 启动类

| 文件 | 作用 |
|------|------|
| `backend/src/.../ClubApplication.java` | SpringBoot 启动入口 |

### 2.3 通用模块（common）

| 文件 | 作用 |
|------|------|
| `common/Result.java` | 统一响应体：`{code, msg, data}` 结构 |
| `common/JwtUtil.java` | JWT 工具类：生成 Token、验证 Token、提取 userId/role |
| `common/GlobalExceptionHandler.java` | 全局异常处理：捕获运行时异常、参数校验异常并返回统一格式 |

### 2.4 安全配置（config）

| 文件 | 作用 |
|------|------|
| `config/SecurityConfig.java` | SpringSecurity + JWT 整合：放行登录注册、JWT 过滤器、CORS 信任 localhost:5173、RBAC 角色权限 |

### 2.5 控制器层（controller）— 3 个

| 文件 | REST 路径 | 功能说明 |
|------|-----------|----------|
| `controller/AuthController.java` | `/api/user` | 登录(`POST /login`)、注册(`POST /register`)、获取当前用户(`GET /info`) |
| `controller/BusinessController.java` | `/api` | 核心业务（40+ 接口）：社团CRUD+审批、活动发布+报名+签到、场地预约+审批、物资借用+归还+审批、经费申请+审批、公告发布+置顶、数据统计、用户管理（启用/禁用） |
| `controller/AiPlanController.java` | `/api/ai` | AI 智能策划接口：调用 LLM 生成活动方案 |

| `entity/Activity.java` | `activity` | 活动信息 |
| `entity/ActivityEnroll.java` | `activity_enroll` | 活动报名记录 |
| `entity/ActivityCheckin.java` | `activity_checkin` | 活动签到记录 |
| `entity/Venue.java` | `venue` | 场地信息 |
| `entity/VenueBooking.java` | `venue_booking` | 场地预约记录 |
| `entity/ResourceItem.java` | `resource_item` | 物资信息 |
| `entity/ResourceBorrow.java` | `resource_borrow` | 物资借用记录 |
| `entity/FundRecord.java` | `fund_record` | 经费收支记录 |
| `entity/SysAnnouncement.java` | `sys_announcement` | 系统公告 |

### 2.7 数据访问层（mapper）— 14 个

| 文件 | 对应实体 | 说明 |
|------|---------|------|
| `mapper/UserMapper.java` | User | 用户数据访问 |
| `mapper/ClubMapper.java` | Club | 社团数据访问 |
| `mapper/ClubMemberMapper.java` | ClubMember | 社团成员数据访问 |
| `mapper/ActivityMapper.java` | Activity | 活动数据访问 |
| `mapper/ActivityEnrollMapper.java` | ActivityEnroll | 报名数据访问 |
| `mapper/ActivityCheckinMapper.java` | ActivityCheckin | 签到数据访问 |
| `mapper/VenueMapper.java` | Venue | 场地数据访问 |
| `mapper/VenueBookingMapper.java` | VenueBooking | 场地预约数据访问 |
| `mapper/ResourceItemMapper.java` | ResourceItem | 物资数据访问 |
| `mapper/ResourceBorrowMapper.java` | ResourceBorrow | 物资借用数据访问 |
| `mapper/FundRecordMapper.java` | FundRecord | 经费数据访问 |
| `mapper/SysAnnouncementMapper.java` | SysAnnouncement | 公告数据访问 |

### 2.8 业务逻辑层（service）

| 文件 | 作用 |
|------|------|
| `service/UserService.java` | 用户服务：登录验证(BCrypt)、注册(密码加密)、用户信息查询 |

### 2.9 LLM/AI 模块（llm）— 创新模块 ⭐

| 文件 | 作用 |
|------|------|
| `llm/ClaudeClient.java` | LLM API 客户端：支持 DeepSeek/Anthropic 双 Provider，3次指数退避重试(2s→4s→8s)，120秒读取超时 |
| `llm/AgentOrchestrator.java` | Agent 编排器：构造 System Prompt → 调用 LLM → 解析响应为结构化策划方案 |
| `llm/MermaidUtil.java` | Mermaid 工具类：图表代码清洗、语法校验（预留扩展） |
| `llm/LangfuseClient.java` | Langfuse 可观测性客户端：LLM 调用追踪日志（可选启用） |

### 2.10 配置文件

| 文件 | 作用 |
|------|------|
| `backend/src/main/resources/application.yml` | 核心配置：数据库连接(MySQL root/root)、JWT密钥+过期时间、LLM Provider选择+API Key、Langfuse开关 |

## 🟡 三、数据库（Database）— `sql/`

**数据库**：MySQL 8.0 (club_management) · **ORM**：MyBatis-Plus (后端)

| 文件 | 作用 |
|------|------|
| `sql/init.sql` | 完整建库建表脚本：1个数据库 + 14张表 + 索引 + 默认数据（admin/president01/student01/teacher01 等演示账号） |

### 14 张数据表一览

| # | 表名 | 说明 | 关键字段 |
|---|------|------|----------|
| 1 | `sys_user` | 系统用户 | id, username, password(BCrypt), role(4种), college, status |
| 2 | `club` | 社团 | id, name, category, president_id, member_count, status |
| 3 | `club_member` | 社团成员 | id, club_id, user_id, role(PRESIDENT/MEMBER), status |
| 4 | `activity` | 活动 | id, title, club_id, status(状态机), max_participants, checkin_code |
| 5 | `activity_enroll` | 活动报名 | id, activity_id, user_id, status |
| 6 | `activity_checkin` | 活动签到 | id, activity_id, user_id, checkin_time |
| 7 | `venue` | 场地 | id, name, location, capacity, facilities |
| 8 | `venue_booking` | 场地预约 | id, venue_id, booking_date, start_time, end_time, status |
| 9 | `resource_item` | 物资 | id, name, category, total, available, unit |
| 10 | `resource_borrow` | 物资借用 | id, item_id, user_id, quantity, plan_return_date, status |
| 11 | `fund_record` | 经费记录 | id, club_id, type(INCOME/EXPENSE), amount, category, status |
| 12 | `sys_announcement` | 系统公告 | id, title, content, is_top, created_by |
| 13 | `sys_notification` | 系统通知 | id, user_id, title, content, is_read |
| 14 | `llm_call_log` | AI调用日志 | id, agent_type, prompt_hash, tokens, duration_ms, success |
